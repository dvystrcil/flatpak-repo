# Flatpak repository for MekHQ and related apps

## Files

```
flatpack-repo/
├── README.md
├── com.megamek.mekhq.json      # Flatpak manifest for MekHQ
├── mekhq-wrapper.sh            # Wrapper script to launch MekHQ
├── .github/
│   └── workflows/
│       ├── build.yml           # Main CI: build + publish to GitHub Pages
│       └── pr-validation.yml   # PR-only build validation
└── repo/                       # Auto-generated: Flatpak OSTree repo (git-ignored)
```

## Workflow Overview

| Trigger | Runs |
|---------|------|
| `main` branch push | Builds & publishes to `dvystrcil.github.io/flatpack-repo/` |
| Pull request | Validates build succeeds (no publish) |

## Setup Steps (for you)

1. **Enable GitHub Pages**
   - Go to repo → Settings → Pages
   - Source: `gh-pages` branch → Save

2. **Add your remote**
   ```bash
   cd flatpack-repo
   git remote add origin https://github.com/dvystrcil/flatpack-repo.git
   git add .
   git commit -m "Add MekHQ Flatpak and CI"
   git push -u origin main
   ```

3. **Verify workflow**
   - Check the **Actions** tab in GitHub
   - You'll see the CI build the Flatpak

4. **Test locally (optional)**
   ```bash
   flatpak remote-add --user mekhq-repo https://dvystrcil.github.io/flatpack-repo/
   flatpak install --user mekhq-repo com.megamek.mekhq
   flatpak run com.megamek.mekhq
   ```

## Notes

- `repo/` is git-ignored — it's auto-generated by CI
- Only `com.megamek.mekhq` is included for now. To add more apps, create new `.json` manifests and update CI accordingly.
